Problem Statement

Why Invoxa?

Many individuals and small businesses struggle with scattered invoices across paper, PDFs, images, and emails.

Scattered Documents

Invoices spread across paper, emails, PDFs, and phone photos with no central repository.

Time-Consuming Search

Finding a specific invoice means digging through folders, drawers, and inboxes manually.

No Expense Tracking

No visibility into spending patterns, monthly trends, or category breakdowns.

Human Errors

Manual data entry leads to errors in amounts, dates, and duplicate records.

What Invoxa aims to deliver for users and small businesses.

Centralize Invoices

Request Flow

How It Works

The complete lifecycle of a user request through the Invoxa system.

User
Clicks link or\nsubmits form
Browser
HTTP Request\nGET / POST
PHP Page
Route & session\ncheck (auth.php)
Backend
functions.php\nCRUD & queries
MySQL
PDO prepared\nstatements
Response
HTML + JS\nrendered page

Standard Page Request

1 User clicks a link (e.g. "Invoices")
2 Browser sends GET request to invoices.php
3 PHP calls requireLogin() to check session
4 functions.php queries MySQL via PDO
5 PHP renders HTML with data, sends response
6 Browser renders page + JS (Chart.js, filters)

AJAX File Upload (Async)

1 User drags a PDF onto the drop zone
2 JS sends XHR POST to api-upload.php
3 Server saves file to uploads/ directory
4 OCR simulation extracts vendor, amount, date
5 Server returns JSON with extracted data
6 JS auto-fills form fields + shows OCR badges

Security

Security Measures

Key security practices implemented throughout the application.

Password Hashing

All passwords are hashed using PHP's bcrypt algorithm (PASSWORD_DEFAULT). Plaintext passwords are never stored.

Session Protection

Secure session management with HTTP-only cookies, session destruction on logout, and route protection via requireLogin().

SQL Injection Prevention

All database queries use PDO prepared statements with parameterized bindings, preventing SQL injection attacks.

XSS Prevention

All user-generated content is escaped using htmlspecialchars() before rendering in HTML templates.
